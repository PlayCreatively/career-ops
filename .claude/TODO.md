- [DONE] is the JD stored? Do we have access to it for filtering?
  → No, the JD body is not stored. Each snapshot row is {title,url,company,location,companyUrl?} + now the optional {postedDate,workMode,department}. The full description IS fetched transiently by check-liveness.mjs during the liveness pass but discarded — never persisted. Content-based filtering would require tier-2 above (capture descriptionHtml/Plain into the snapshot). BUT the new structured fields are now first-class FILTER dimensions without storing the body: `fieldText` in rank.mjs + site/rank.browser.js (parity) gained `department` (text) and `workmode` (matches the 'remote'/'hybrid'/'onsite' token; unknown → '' → falls to the group catch-all). This also powers the scan gate (groupDropReason→matchGroup) — a portals.yml `targeting.groups` entry with field: department/workmode just works. Board group editor exposes both; department folded into free-text search + the `any` field. Verified live: dept-exclude dropped art+audio; workmode-exclude-onsite dropped all onsite Larian roles. **Multi-source fields:** a group's `field` may now be an ARRAY of sources (e.g. `[location, workmode]`) — fieldText joins them into one combined string (generalises the fixed `any`). This is what lets cross-field filters survive the location strip: "US-only remote" (regex needs US-from-location AND remote-from-workmode) reads `[location, workmode]` so the combined text ≈ the original un-stripped location, matching identically. Board group editor's field picker is now multi-select checkboxes (was a single dropdown); sanitizeField accepts string|array, collapses a 1-element array to a string. Migrated: targeting.default.yml "Mode" group → `[workmode, location]` (+ added Hybrid filter); personal portals.yml "Region" group → `[location, workmode]`. scan.mjs groupDropReason handles array fields for the drop counter. Tests added (test-all #17): splitLocationMode shapes, normalizeWorkMode, multi-source fieldText, cross-field exclude. 182 pass (only Go dashboard build fails — Go not installed).
